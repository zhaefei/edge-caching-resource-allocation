# Experiment Plan v2

This document lists the final experiment set planned for the v2.0 portfolio
version of the project. Each experiment should be reproducible from scripts,
use fixed random seeds where possible, and save CSV results under
`results/data/` and figures under `results/figures/`.

## 1. Cache Capacity Experiment

Purpose:

- Evaluate how edge cache size affects average latency, cache hit ratio, and
  backhaul load.

Expected interpretation:

- Larger cache capacity should generally improve hit ratio and reduce backhaul
  traffic.
- Caching-aware policies should outperform random caching when content requests
  follow a Zipf distribution.

## 2. User Density Experiment

Purpose:

- Study how the number of users changes wireless rate and latency under shared
  edge-server bandwidth.

Expected interpretation:

- As user density increases, bandwidth per user may decrease, increasing
  wireless transmission delay.
- Demand-aware allocation may improve request-weighted latency when traffic
  demand is uneven.

## 3. Zipf Alpha Experiment

Purpose:

- Evaluate how content popularity skew affects caching performance.

Expected interpretation:

- Higher Zipf alpha means popular files dominate requests, making caching more
  effective.
- Low Zipf alpha makes requests more uniform, reducing the advantage of simple
  popularity-based caching.

## 4. Spatial Locality Experiment

Purpose:

- Study how server-specific local preferences change the value of local
  popularity caching.

Expected interpretation:

- When spatial locality is stronger, local popularity caching should become more
  useful relative to a single global popularity ranking.

## 5. Wireless Channel Sensitivity Experiment

Purpose:

- Compare latency and wireless rate under different channel-model assumptions or
  channel parameters.

Planned variables may include:

- Path-loss exponent
- Optional fading enabled/disabled
- Fading strength or random-seed sensitivity

Expected interpretation:

- Worse channel conditions should reduce wireless rate and increase wireless
  transmission delay.
- Channel sensitivity helps separate wireless bottlenecks from caching and
  backhaul effects.

## 6. MAB Caching Comparison Experiment

Purpose:

- Compare a simple Multi-Armed Bandit caching policy against the existing
  random, popularity-based, local popularity, and greedy latency-aware caching
  strategies.

Expected interpretation:

- The MAB policy should be presented as a lightweight adaptive baseline, not as a
  novel or optimal caching algorithm.
- Results should focus on whether online request feedback can approach or
  improve simple static baselines under the simplified simulator assumptions.

## 7. Multi-Seed v2 Summary Experiment

Purpose:

- Repeat the final v2 experiment set across multiple random seeds and summarize
  mean and standard deviation.

Expected interpretation:

- Multi-seed results should distinguish stable trends from single-seed artifacts.
- Final claims in README and report should use numbers generated by these
  scripts, not manually invented values.

## Reporting Requirements

Final summaries should include:

- Average latency
- Median and 95th percentile latency
- Cache hit ratio
- Backhaul traffic or load ratio
- Average wireless transmission rate
- Wireless/backhaul latency breakdown
- Jain's bandwidth fairness index where bandwidth allocation is compared

All reported numerical results should come from generated CSV files.
